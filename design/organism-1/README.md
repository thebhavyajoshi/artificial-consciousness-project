# Organism-1

## Purpose

Organism-1 is the project's first system designed specifically for artificial identity-continuity experiments.

It will be built only after Organism-0 provides a deterministic, tested baseline for embodiment, homeostasis, partial observability, logging, and controlled body changes.

Organism-1 is not intended to prove subjective consciousness. It is intended to test which architectural components are causally necessary for an artificial agent to remain a coherent individual across time and intervention.

## Relationship to Organism-0

### Organism-0

Replication and engineering baseline:

- embodied 2D environment;
- energy, structural integrity, and fatigue;
- partial observations;
- homeostatic valuation;
- recurrent control;
- internal-state prediction;
- deterministic seeds and logs; and
- simple body-change experiments.

### Organism-1

Identity-continuity research system:

- autobiographical memory;
- explicit memory provenance;
- predictive body and capability model;
- persistent self-model;
- identity confidence and uncertainty;
- exact checkpointing and cloning;
- body replacement;
- memory transplantation;
- false-memory insertion;
- selective memory deletion;
- interoceptive corruption; and
- modular lesions and restoration.

## Required modules

### 1. Body state

The current embodied condition, including:

- energy;
- integrity;
- fatigue;
- position and orientation;
- sensor configuration;
- action capabilities;
- resource requirements; and
- vulnerability profile.

### 2. Episodic autobiographical memory

Each self-related event should record:

- unique event ID;
- agent and lineage ID;
- world tick;
- body ID;
- checkpoint ancestry;
- observation;
- action;
- predicted outcome;
- actual outcome;
- internal state before and after;
- salience;
- confidence;
- causal links; and
- provenance.

### 3. Semantic self-knowledge

Consolidated beliefs such as:

- “I require energy to remain viable”;
- “my forward movement usually costs this much energy”;
- “my current body cannot cross walls”;
- “hazards of this type damage my integrity”; and
- “this memory was imported rather than directly experienced.”

### 4. Predictive self-model

The model should predict:

- the next internal state after an action;
- whether an action is physically possible;
- expected sensor changes;
- expected homeostatic consequences;
- confidence in the prediction; and
- whether prediction error is more likely caused by self-change or world-change.

### 5. Identity state

A structured, inspectable representation containing:

- lineage ID;
- current body ID;
- checkpoint ancestry;
- continuity confidence;
- memory-ownership confidence;
- stable preferences;
- current goals;
- capability beliefs; and
- unresolved self-related contradictions.

This structure is an experimental model, not proof that the system possesses a subjective self.

### 6. Intervention system

The platform must support reproducible operations such as:

```python
clone = organism.clone(new_agent_id="copy-b")
organism.replace_body(body_config)
organism.import_memories(memory_archive, provenance="imported")
organism.insert_false_memory(event)
organism.delete_memories(query)
organism.corrupt_interoception(variable="energy", bias=0.30)
organism.disable_module("self_model")
organism.restore_checkpoint(checkpoint_id)
```

### 7. Checkpoint and lineage system

Every checkpoint must preserve:

- software and configuration version;
- environment seed and state;
- agent lineage;
- body state;
- policy parameters;
- recurrent state;
- episodic memory;
- semantic memory;
- self-model state;
- preferences and goals;
- intervention history; and
- random-number-generator states.

## Flagship experiments

1. Exact cloning followed by identical experience.
2. Exact cloning followed by divergent experience.
3. Memory transplantation between bodies.
4. Body replacement with preserved memory.
5. False-memory insertion.
6. Episodic versus semantic memory deletion.
7. Interoceptive corruption.
8. Self-model lesion.
9. Global-integration lesion.
10. Recovery after restoring a previous checkpoint.

## Comparison architectures

Each experiment should compare multiple configurations:

1. recurrent policy only;
2. recurrent policy plus ordinary memory;
3. homeostatic agent;
4. homeostatic agent plus self-model;
5. homeostatic agent plus autobiographical memory;
6. homeostatic agent plus self-model and autobiographical memory;
7. complete Organism-1 architecture; and
8. relevant lesion variants.

## Outputs

Organism-1 should generate:

- inspectable event histories;
- lineage graphs;
- identity profiles;
- memory-provenance reports;
- prediction-error traces;
- copy-divergence curves;
- intervention-effect reports;
- reproducible checkpoints; and
- human-readable experiment summaries.

## Completion condition

Organism-1 will be considered technically ready for the first Identity Continuity Benchmark when:

- exact checkpoints reproduce identical trajectories;
- copies remain identical under identical post-copy experiences;
- controlled divergent experiences produce traceable divergence;
- body and memory interventions can be applied independently;
- provenance is preserved through memory transfer;
- all modules can be ablated without corrupting unrelated state;
- benchmark metrics are computed automatically; and
- experiments run across multiple seeds without manual intervention.

## Scientific boundary

A system passing these experiments may demonstrate persistent self-related computation, memory ownership inference, coherent adaptation, and identity-related functions. It would not by itself demonstrate subjective experience, moral status, or personhood.
